FIREBASE 설정 가이드

공수픽 — Firebase Analytics + Crashlytics 활성화

Firebase 프로젝트 생성 → flutterfire configure → 빌드 한 번이면 끝

최종 수정: 2026-05-09 · 코드는 이미 Firebase 호출 가능 상태. 아래 두 단계만 진행하시면 됩니다.

현재 상태 Firebase 미설정 → 앱은 정상 작동, Analytics 이벤트는 NoopAnalyticsService(콘솔에 debug print만)로 처리. 아래 단계를 마치면 자동으로 실 Firebase로 전환됩니다.

STEP 1 · Firebase 프로젝트 만들기

1Firebase Console 진입

Firebase Console →

  1. 프로젝트 만들기 클릭
  2. 프로젝트 이름: gongsupick (자유)
  3. Google Analytics: 사용 선택 → 계정 선택/생성
  4. 국가: 대한민국 → 만들기

생성에 약 1분 걸립니다.

STEP 2 · flutterfire CLI로 자동 연결 (가장 쉬움)

2로컬에서 명령 한 번

아래 명령은 아래 작업을 자동으로 처리합니다:

  • Android 앱 등록
  • android/app/google-services.json 다운로드
  • lib/firebase_options.dart 생성
  • Gradle 플러그인 자동 설정
# 1) flutterfire CLI 설치 (한 번만)
dart pub global activate flutterfire_cli

# 2) PATH에 pub bin 추가 (zshrc/bashrc에 영구 추가 권장)
export PATH="$PATH":"$HOME/.pub-cache/bin"

# 3) 프로젝트 루트에서 실행
cd /Users/dev/Documents/flutter/GongsuPick
flutterfire configure

대화형 입력:

  • Select a Firebase project: 방금 만든 gongsupick 선택
  • Which platforms: Android만 체크 (iOS 나중에)
  • 완료까지 약 30초

STEP 3 · main.dart 한 줄 수정

3options 명시

flutterfire가 만든 lib/firebase_options.dart를 사용하도록 main.dart 수정:

// lib/main.dart 상단 import 추가
import 'firebase_options.dart';

// main() 안의 Firebase.initializeApp() 호출을 다음과 같이 수정:
await Firebase.initializeApp(
  options: DefaultFirebaseOptions.currentPlatform,
);

이 변경 없이도 google-services.json만 있으면 Android에서는 동작하지만, options 명시가 정석.

STEP 4 · Crashlytics Gradle 플러그인 추가

4android/build.gradle.kts

// android/build.gradle.kts (project-level)
plugins {
    // ... 기존 플러그인 ...
    id("com.google.gms.google-services") version "4.4.2" apply false
    id("com.google.firebase.crashlytics") version "3.0.2" apply false
}

5android/app/build.gradle.kts

// android/app/build.gradle.kts 상단 plugins 블록에 추가
plugins {
    id("com.android.application")
    id("kotlin-android")
    id("dev.flutter.flutter-gradle-plugin")
    // 추가:
    id("com.google.gms.google-services")
    id("com.google.firebase.crashlytics")
}

flutterfire configure가 STEP 4의 첫 부분(google-services 플러그인)은 자동으로 추가하지만, Crashlytics 플러그인은 수동으로 추가해야 합니다.

STEP 5 · 빌드 + 검증

6릴리즈 모드에서만 Crashlytics 동작

# 빌드
flutter build appbundle --release

# 또는 디버그로 동작 확인 (Crashlytics는 release만 보고)
flutter run

7Firebase Console에서 확인

  • Analytics → Realtime: 앱 실행 시 1~2분 내 사용자 수 증가
  • Analytics → DebugView: 디버그 빌드의 이벤트가 즉시 표시 (디바이스에 adb shell setprop debug.firebase.analytics.app kr.co.blot.gongsupick 한 번 실행)
  • Crashlytics: 첫 크래시 후 자동 등록 (신규 프로젝트는 1회 sample crash 권유 — 무시 OK)

코드는 이미 무엇을 보내나요?

이벤트발생 시점파라미터
app_open앱 시작(Firebase 기본)
entry_saved공수 입력 시트 저장gongsu, is_settled, has_group, is_holiday
job_clicked일감/북마크 클릭source_type, position
webview_closedWebView 닫기dwell_seconds
backup_exportJSON 내보내기 성공(없음)

광고 수익 최적화에 가장 중요한 5개로 압축. 이벤트 수가 적을수록 Firebase Analytics가 패턴 학습이 쉽습니다.

iOS 추가 시 (나중에)

iOS 앱 등록 후 같은 flutterfire configure 명령에서 iOS도 체크하면 자동으로 GoogleService-Info.plist가 ios/Runner/에 들어가고 firebase_options.dart에 iOS 옵션이 추가됩니다. 추가 코드 변경 없음.
