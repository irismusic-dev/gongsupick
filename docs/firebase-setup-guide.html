FIREBASE 설정 가이드

공수픽 — Firebase Analytics + Crashlytics 활성화

Firebase 프로젝트 생성 → flutterfire configure → 빌드 한 번이면 끝

최종 수정: 2026-05-09 · 코드는 이미 Firebase 호출 가능 상태. 아래 두 단계만 진행하시면 됩니다.

현재 상태 Firebase 미설정 → 앱은 정상 작동, Analytics 이벤트는 NoopAnalyticsService(콘솔에 debug print만)로 처리. 아래 단계를 마치면 자동으로 실 Firebase로 전환됩니다.

STEP 1 · Firebase 프로젝트 만들기

1Firebase Console 진입

Firebase Console →

  1. 프로젝트 만들기 클릭
  2. 프로젝트 이름: gongsupick (자유)
  3. Google Analytics: 사용 선택 → 계정 선택/생성
  4. 국가: 대한민국 → 만들기

생성에 약 1분 걸립니다.

STEP 2 · CLI 도구 설치

2flutterfire + Firebase CLI 두 개 모두 필요

flutterfire는 dart pub으로, Firebase CLI는 npm으로 설치 (이미 완료된 항목은 재실행 안전).

# flutterfire CLI (dart 패키지)
dart pub global activate flutterfire_cli

# Firebase CLI (npm 패키지) — flutterfire가 내부적으로 사용
npm install -g firebase-tools

# PATH 추가 (~/.zshrc 또는 ~/.bashrc)
export PATH="$PATH":"$HOME/.pub-cache/bin"
현재 환경에 둘 다 이미 설치되어 있습니다 (개발 환경 셋업 단계에서 처리됨). 설치 명령을 다시 실행해도 안전합니다.

STEP 3 · Firebase 로그인 + 프로젝트 연결

3OAuth 로그인 한 번

# 본인 Google 계정으로 로그인 (브라우저 OAuth)
firebase login

# 프로젝트 루트에서 실행
cd /Users/dev/Documents/flutter/GongsuPick
flutterfire configure --project=gongsupick --platforms=android

실행 시 자동으로 처리되는 작업:

  • Android 앱 등록 (kr.co.blot.gongsupick)
  • android/app/google-services.json 다운로드
  • lib/firebase_options.dart 생성 (placeholder를 실제 값으로 덮어씀)
  • Gradle com.google.gms.google-services 플러그인 자동 추가

완료까지 약 30초.

STEP 4 · main.dart는 이미 적용됨

✅ 코드 작업 완료 main.dart는 이미 다음과 같이 수정되어 있어 STEP 3 실행 후 자동으로 동작합니다: 
import 'firebase_options.dart';
// ...
await Firebase.initializeApp(
  options: DefaultFirebaseOptions.currentPlatform,
);
placeholder firebase_options.dart는 throws → main.dart의 try/catch가 잡아 Noop 폴백 → STEP 3 후 실제 옵션으로 자동 교체.

STEP 5 · Crashlytics Gradle 플러그인 추가

5android/build.gradle.kts

// android/build.gradle.kts (project-level)
plugins {
    // ... 기존 플러그인 ...
    id("com.google.gms.google-services") version "4.4.2" apply false
    id("com.google.firebase.crashlytics") version "3.0.2" apply false
}

5android/app/build.gradle.kts

// android/app/build.gradle.kts 상단 plugins 블록에 추가
plugins {
    id("com.android.application")
    id("kotlin-android")
    id("dev.flutter.flutter-gradle-plugin")
    // 추가:
    id("com.google.gms.google-services")
    id("com.google.firebase.crashlytics")
}

flutterfire configure가 STEP 4의 첫 부분(google-services 플러그인)은 자동으로 추가하지만, Crashlytics 플러그인은 수동으로 추가해야 합니다.

STEP 6 · 푸시 알림 (FCM) 설정 + 사용

7코드는 준비됨 — 콘솔에서 메시지만 보내면 됨

현재 앱은 다음과 같이 구성되어 있습니다:

  • firebase_messaging 패키지 통합 완료
  • 앱 첫 실행 시 알림 권한 요청 (Android 13+ POST_NOTIFICATIONS, iOS APNs)
  • all_users 토픽 자동 구독 — 이 토픽에 메시지 보내면 모든 사용자에게 도착
  • foreground/background 모두 처리: 백그라운드는 시스템 알림으로 자동 표시

관리자 측 — Firebase Console에서 푸시 보내기

  1. Firebase Console → Messaging
  2. 새 캠페인 만들기 → 알림
  3. 제목/본문 입력 (예: "새 일감 정보가 업데이트됐어요")
  4. 대상: 주제(Topic) → all_users
  5. 예약 또는 즉시 전송

활용 사례

  • 새 버전 출시 알림
  • 일감 정보 업데이트 (주 1~2회 권장)
  • 이벤트/할인 안내
  • 중요 공지 (서비스 정책 변경 등)
주의: 너무 잦은 푸시는 설정 → 알림 끄기로 이어지고, 결국 토픽 unsubscribe됨. 주 1~2회를 넘지 않도록 권장. 사용자 ID별 push가 필요해지면 향후 token 저장 + 서버 발송 로직 추가 가능.

STEP 7 · 빌드 + 검증

8릴리즈 모드에서만 Crashlytics 동작

# 빌드
flutter build appbundle --release

# 또는 디버그로 동작 확인 (Crashlytics는 release만 보고)
flutter run

9Firebase Console에서 확인

  • Analytics → Realtime: 앱 실행 시 1~2분 내 사용자 수 증가
  • Analytics → DebugView: 디버그 빌드의 이벤트가 즉시 표시 (디바이스에 adb shell setprop debug.firebase.analytics.app kr.co.blot.gongsupick 한 번 실행)
  • Crashlytics: 첫 크래시 후 자동 등록 (신규 프로젝트는 1회 sample crash 권유 — 무시 OK)

iOS 추가 시 (나중에)

iOS 앱 등록 후 같은 flutterfire configure 명령에서 iOS도 체크하면 자동으로 GoogleService-Info.plist가 ios/Runner/에 들어가고 firebase_options.dart에 iOS 옵션이 추가됩니다. 추가 코드 변경 없음.
